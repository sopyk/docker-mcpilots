关于 Docker-MCPilotS

🤔 为什么做这个？

日常使用群晖 NAS，上面跑了几十个 Docker 容器：PT 下载、媒体服务、照片备份、代码仓库……部署新服务的时候、升级更新的时候、甚至正常跑着用着，总会遇到各种问题。

现在 AI Agent 很发达，让 AI 来帮我们管容器是最好的选择，但又不能真的把 SSH 权限交给 Agent，NAS 系统太特殊了，万一出岔子代价太大。所以就想到了 MCP——把 Docker 的管理能力通过 MCP 接口暴露出去，Agent 只能做我们允许的操作。

✨ 功能一览

能做什么

  • 📦 容器管理（查看列表、详情、启停、删除）
  • 📜 容器日志查看（支持时间段筛选）
  • 📊 容器资源占用
  • 🔍 容器诊断（进程、健康、网络、挂载）
  • 🌐 网络拓扑查看
  • 💾 数据卷清单
  • 🖥️ 系统诊断（CPU/内存/磁盘）
  • 🔐 权限控制（三种角色）

限制说明

  • 🚫 不支持 Compose 项目管理
  • 🚫 不支持构建镜像
  • 🚫 不提供 exec 能力
  • 🚫 不支持修改网络配置

🚀 如何在 Agent APP 里配置 MCP

在你的 AI 客户端（Trae、Cursor、Claude Code、OpenClaw 等）的 MCP 配置里添加下面这段代码即可：

{
  "mcpServers": {
    "docker-mcpilots": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/transport-http",
        "http://localhost:8900/mcp",
        "--header",
        "Authorization: sk-dm-你的APIKey"
      ]
    }
  }
}

💡 提示：记得把 http://localhost:8900 和 sk-dm-你的APIKey 换成你实际的地址和 API Key！
