# 权限控制

## 角色说明

| 角色 | 权限 |
|------|------|
| **admin** | 所有操作权限，包括删除容器、exec 命令 |
| **operator** | 启动、停止、重启容器，查看日志和状态（不能删除、不能 exec） |
| **observer** | 只读，只能查看不能操作 |

## 配置方式

编辑 `secrets/auth.yaml`：

```yaml
keys:
  - key: "你的-secret-api-key"
    role: admin
```

## 容器范围控制（Scope）

可以限制某个 API Key 只能操作特定容器：

```yaml
keys:
  - key: "只管下载容器"
    role: operator
    scope:
      include: ["qbittorrent", "aria2*", "transmission"]
      exclude: ["*"]
```

- `include`：允许操作的容器列表，支持通配符 `*`
- `exclude`：禁止操作的容器列表，支持通配符 `*`
- 不配置 scope 则允许操作所有容器

## exec 权限说明

`exec_container` 工具默认**禁用**，需要在设置中手动开启。

启用后，仅 **admin 角色**可以使用。建议配合 scope 限制，只允许对特定工具容器执行 exec，详见 [工具容器](工具容器.md)。

## 安全建议

- 生产环境务必修改默认 API Key
- 根据实际需要分配角色权限，不要都给 admin
- 限制容器 scope 权限，只给必要的权限
- 通过反向代理加 TLS 访问
- 定期检查日志，发现异常及时排查
